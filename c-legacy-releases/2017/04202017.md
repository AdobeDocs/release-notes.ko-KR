**수정 사항, 개선 사항 및 중단 사항**

* 일부 고객의 경우 URL을 URL 대상에 매핑할 때 전체 URL 문자열이 저장되지 않는 문제를 해결했습니다. (AAM-28510)
* RBAC 사용자용 프로필 병합 규칙에서 트레이트를 볼 권한이 없는 사용자에게 페이지를 볼 수 있는 권한이 없다는 경고 메시지 대신 빈 페이지가 표시되는 문제를 해결했습니다. (AAM-28361)
* 일부 고객의 경우 트레이트 목록 페이지에 Data Sources에 대한 깨진 링크가 포함된 문제를 해결했습니다. (AAM-29174)
* 프로필 링크 기능을 개선했습니다. 이제 이 기능은 [Audience Manager UUID](https://marketing.adobe.com/resources/help/en_US/aam/ids-in-aam.html)를 다른 3개의 Audience Manager UUID에 연결하여 클러스터에 총 4개의 Audience Manager UUID가 됩니다. (AAM-28808)
* 이제 다음 API에서 Test Segment ID([SID 및 csegID](https://marketing.adobe.com/resources/help/en_US/aam/ids-in-aam.html))를 쿼리 매개 변수 testSegments= rue로 노출합니다(AAM-27591).
   * ` .../v1/segments`
   * ` .../v1/destinations/<destinationId>/mappings`

* 세그먼트 API 메서드를 Swagger로 병합했습니다. [Audience Manager API 코드 마이그레이션](https://marketing.adobe.com/resources/help/en_US/aam/c_api_swagger.html)을 참조하십시오. (AAM-28601)
* DAID 대상에 대한 Addressable Audience에서 몇 가지 소소한 시각적 버그를 제거했습니다. (AAM-28679)
* 고객이 이미 사용된 사용자 이름으로 회사의 새 사용자를 만들려고 할 때 오류 메시지가 표시되지 않는 사용자 관리 페이지 문제를 해결했습니다. (AAM-28945)
* 비밀번호 재설정 모드 방식, Audience Marketplace 및 사용자 인터페이스에서 목록 페이지의 열을 정렬할 때 잘못된 아이콘이 표시되는 시각적 버그를 해결했습니다. (AAM-28953)
* Audience Marketplace 최종 사용자 ID를 회사에 추가할 수 없는 Audience Manager Admin 인터페이스 문제를 해결했습니다. (AAM-28953)

## [!DNL Social]

**수정 사항**

* 게시 워크플로우: 승인 워크플로우를 거치지 않은 사용자가 여전히 승인 알림을 받는 문제를 해결했습니다. 워크플로우의 승인자인 사용자만 게시물 승인 메시지를 받게 됩니다.
* 어디서나 게시: 사진 또는 동영상 업로드가 특정 플랫폼의 API 크기 제한을 초과할 때 사용자에게 경고가 표시되지 않는 문제를 해결했습니다.
* 어디서나 게시: Facebook에 있는 타깃팅된 사진 앨범 게시물이 사진이 없는 채로 게시되던 문제를 수정했습니다.
* 컨텐츠 달력: 사용자가 예정된 하위 게시물의 시간을 수정할 수 없는 문제를 해결했습니다.
* 컨텐츠 달력: 비디오를 Twitter에 게시할 때 이모티콘과 텍스트가 표시되지 않던 문제를 수정했습니다.
* 보고서: 중재 개요 보고서를 다운로드할 때 오류가 발생하는 문제를 해결했습니다.
* Facebook 속성 세부 사항: Facebook 속성에 대한 지리적 위치별 팬 데이터를 획득하는 것과 관련된 문제를 수정했습니다.
* 관리: 사용자가 속성 또는 작성자에 대한 여러 줄 설명을 작성할 수 없던 문제를 수정했습니다.
* 관리: [작성자 세부 정보] 페이지에 작성자와의 상호 작용에 대한 전체 기록이 표시되지 않던 문제를 수정했습니다.
* 속성: 속성에 권한이 없어질 경우 사용자가 알림 이메일을 받지 못하던 문제를 수정했습니다.
* 소셜 태그: LinkedIn 게시물에 적용된 일부 태그가 게시물 분석에서 표시되지 않던 문제를 수정했습니다.
