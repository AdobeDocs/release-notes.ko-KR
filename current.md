### Adobe Analytics 및 Customer Journey Analytics의 수정 사항 {#aa-fixes}

* 대상 ID가 차원 항목에서 누락되었던 Analysis Workspace 문제가 해결되었습니다. (AN-262038; AN-279315)
* 사용자가 저장된 을 로드할 수 없는 문제를 해결했습니다 [!DNL Target] 프로젝트를 만들 수 있습니다. (AN-277461) AN-275825; (AN-266397)
* UI에 활성화되지 않은 기능이 표시되는 문제가 수정되었습니다. (AN-262006)
* 작업 공간의 날짜 필드를 사용하여 날짜를 변경할 때 발생하는 문제를 수정했습니다. 그 결과 [!UICONTROL 종료 시간] 오후 11:59에서 오전 12:00으로 변경 (AN-277269; AN-277481)
* 이미 로드된 세그먼트에 새 세그먼트를 추가할 때 세그먼트 UI가 중단되는 문제를 수정했습니다. (AN-260827)
* 사용자가 공유 Workspace 프로젝트에 액세스할 수 없는 문제를 해결했습니다. (AN-267529)
* 롤링 날짜 범위의 시작 날짜가 종료 날짜보다 늦은 시기를 표시하는 오류 메시지가 추가되었습니다. (AN-270488)
* 다양한 데이터 피드 문제가 해결되었습니다. (AN-275876) AN-270512; AN-277284; AN-277290; AN-274893; AN-274606; (AN-269651)
* 그래프의 날짜 범위에서 테이블의 날짜 필터를 무시하는 문제를 해결했습니다. (AN-263999)
* 일광 절약 시간제로 인해 예약된 보고서가 일찍 전송되던 문제를 해결했습니다. (AN-276410; AN-276305)
* 프로젝트 다운로드에서 `.csv` 작업 공간에서 파일이 실패했습니다. (AN-275834)

      * 이제 Git 커밋 ID가 파이프라인 실행 세부 사항에 표시되므로 빌드된 코드를 쉽게 추적할 수 있습니다.
      * [!UICONTROL 프로그램 만들기] 는 이제 공개적으로 노출된 API를 통해 사용할 수 있습니다.
      * [!UICONTROL 환경 만들기] 는 이제 공개적으로 노출된 API를 통해 사용할 수 있습니다.
      * 다음 `x-request-id` 이제 응답 헤더가 의 API Playground에 표시됩니다. [www.adobe.io](https://www.adobe.io/). 이 헤더는 문제 해결을 위해 고객 지원 문제를 제출할 때 유용합니다.
      * 새로운 [!UICONTROL 활동] 페이지는 파이프라인 및 코드 실행과 같은 활동을 연관된 세부 정보와 함께 볼 수 있는 페이지에서 사용할 수 있습니다. 시간이 지나면서 이 페이지에 나열된 활동은 제공된 세부 정보와 함께 범위가 확장됩니다.
      * 새로운 [!UICONTROL 파이프라인] 이제 세부 사항 요약을 쉽게 볼 수 있도록 상태 팝오버가 포함된 페이지를 사용할 수 있습니다. 다음을 볼 수 있습니다 [!UICONTROL 파이프라인] 은 관련 세부 정보와 함께 을 실행합니다.
      * 다음 [!UICONTROL 파이프라인 편집] 이제 API는 배포 단계에서 사용되는 환경 변경을 지원합니다.
      * 의 최적화 [!DNL OakPal] 이제 큰 패키지에 대해 검색 프로세스를 사용할 수 있습니다.
      * 이제 품질 문제 CSV 파일에 각 품질 문제에 대한 타임스탬프가 포함됩니다.
